GitHub release analytics for project maintainers

Maintainers often need to understand how public release assets are used, but GitHub release pages show raw asset counters without a broader report. GRA Release Radar reads public GitHub release data in the browser and summarizes download totals, release cadence, package coverage, and platform adoption.

Questions this use case answers

  • Which release assets receive most downloads?
  • Are Windows, macOS, Linux, ARM, x64, zip, tar.gz, deb, rpm, dmg, exe, and checksum assets represented consistently?
  • Did a recent release change download behavior?
  • Are prereleases hiding or distorting stable release trends?
  • Are downloads concentrated in a few assets or spread across the package matrix?

How to start

Open GRA Release Radar and enter a public repository such as vercel/next.js or tauri-apps/tauri. If GitHub rate limits the browser session, wait for the reset time or apply a read-only GitHub token in the browser.

Important limits

GRA does not count GitHub source archive downloads because GitHub does not expose those values through release asset download counters. It does not operate a hosted analysis API or private telemetry backend.
